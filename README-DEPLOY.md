# Deployment Guide: MovieShark / BookMyShow

This guide provides instructions to deploy your MovieShark (BookMyShow) application with the **Frontend on Vercel** and the **Backend on Render**.

---

## Prerequisites: Cloud Services Setup

Because the application relies on **MySQL** and **Elasticsearch**, you need hosted cloud versions of these services before deploying the backend.

### 1. Hosted MySQL Database (e.g., Aiven or Railway)
Choose a platform like [Aiven.io](https://aiven.io/) or [Railway](https://railway.app/):
1. Create a free account.
2. Spin up a new **MySQL** database instance.
3. Once active, retrieve the connection URL, database username, and password.
   * *Example URL:* `jdbc:mysql://mysql-instance-hostname.aivencloud.com:port/defaultdb?ssl-mode=REQUIRED`

### 2. Hosted Elasticsearch Instance (e.g., Bonsai or Elastic Cloud)
You can use [Bonsai.io](https://bonsai.io/) (it has a free/hobby tier) or [Elastic Cloud](https://cloud.elastic.co/):
1. Sign up for Bonsai.io.
2. Create a free-tier cluster.
3. Copy the **Elasticsearch Access URL** (usually looks like `https://username:password@some-cluster.bonsai.net`).

## Step 1: Deploy Backend to Render

1. Go to [Render.com](https://render.com/) and sign in.
2. Click **New** -> **Web Service**.
3. Connect your GitHub Repository containing this project.
4. Set the following fields:
   * **Name:** `movieshark-backend`
   * **Region:** Choose the region closest to you or your database.
   * **Branch:** `main` (or `update-1` depending on which branch you merge into).
   * **Root Directory:** *(Leave blank, or set to root `/`)*
   * **Runtime:** `Docker` (Render will automatically detect the `Dockerfile` at the root and build it).
   * **Instance Type:** `Free` (or any tier you prefer).
5. In **Advanced Settings**, add the following **Environment Variables**:

| Key | Value | Description |
| :--- | :--- | :--- |
| `SPRING_DATASOURCE_URL` | `jdbc:mysql://<host>:<port>/<db_name>` | Your cloud MySQL URL |
| `SPRING_DATASOURCE_USERNAME` | `<db_username>` | Your cloud MySQL username |
| `SPRING_DATASOURCE_PASSWORD` | `<db_password>` | Your cloud MySQL password |
| `SPRING_ELASTICSEARCH_URIS` | `https://<bonsai_url_or_elastic_url>` | Your cloud Elasticsearch URL |
| `RAZORPAY_KEY_ID` | `rzp_test_SYJr5BIAdj3CbL` | *(Or your custom Razorpay Key)* |
| `RAZORPAY_KEY_SECRET` | `ON8GM555MzG9Ceya1k0NMrEJ` | *(Or your custom Razorpay Secret)* |
| `SPRING_MAIL_USERNAME` | `manikiphone14email@gmail.com` | *(Or your email username)* |
| `SPRING_MAIL_PASSWORD` | `oxasicxxcteywerw` | *(Or your email app password)* |
| `PORT` | `9099` | Tells Render to expose port 9099 |

6. Click **Deploy Web Service**. Render will start pulling, compiling, and running your Spring Boot application! Note the URL generated by Render once the service is live (e.g., `https://movieshark-backend.onrender.com`).

## Step 2: Deploy Frontend to Vercel

1. Go to [Vercel.com](https://vercel.com/) and sign in.
2. Click **Add New** -> **Project**.
3. Select your GitHub repository.
4. On the **Configure Project** screen:
   * **Project Name:** `movieshark-frontend`
   * **Framework Preset:** `Vite` (Vercel should auto-detect this)
   * **Root Directory:** Click "Edit" and choose **`frontend`** (Crucial: Vercel must build only inside the `frontend` folder).
5. Expand the **Environment Variables** section and add:
   * **Name:** `VITE_API_URL`
   * **Value:** `https://your-backend.onrender.com` (Use the Render URL you copied from Step 1, with **no** trailing slash).
6. Click **Deploy**. Vercel will build the React application and deploy it.

*Note: Client-side routing is handled automatically using the configured `frontend/vercel.json` rewrite rules.*
